# DESIGN.md — 치킨 증바람

## Overview

치킨 증바람은 증강 칼바람 내전의 일정, 결과, 선수 티어를 기록하는 경쟁형 스포츠 보드다. 분위기는 어둡고 정밀하지만 게임 특유의 에너지가 느껴져야 한다. 장식보다 점수, 팀, 승률 같은 정보가 먼저 읽히며 오렌지 포인트는 현재 상태와 핵심 행동에만 사용한다.

## Components

### Header

- 72px 고정 높이, 반투명 다크 배경과 16px 블러.
- 데스크톱에서는 중앙 메뉴, 모바일에서는 하단 고정 내비게이션으로 이동.
- 우측 계정 영역에 사용자 이름과 `관리자` 또는 `일반 계정` 역할 표시.

### Match Card

- 레드 팀 / 점수 또는 VS / 블루 팀의 3열 구조.
- 날짜와 맵은 상단 메타 행, 상태와 MVP는 하단 메타 행에 배치.
- 대표 카드는 그라데이션 표면과 더 큰 팀 이름을 사용하되 구조는 동일하게 유지.

### Tier Board

- 1~4티어를 세로 섹션으로 분리하고 왼쪽에 큰 티어 번호 표시.
- 같은 티어의 선수는 승률 내림차순, 동률이면 승수 내림차순.
- 선수 행에는 순번, 이니셜, 닉네임, 승률만 노출해 빠르게 비교.

### Admin Form

- 대전 예정 페이지의 우측 보조 패널에 배치하고 모바일에서는 목록 아래로 이동.
- 관리자에게만 폼을 렌더링하며 서버에서도 권한을 다시 검증.
- 필수값은 브라우저 기본 검증과 서버 검증을 함께 사용.

## Responsive Rules

- `900px` 이하: 2열 레이아웃을 1열로 전환하고 관리자 패널의 고정을 해제.
- `640px` 이하: 내비게이션을 하단 바로 전환하고 터치 영역을 최소 44px로 유지.
- 티어 선수 그리드는 3열 → 2열 → 1열로 축소.
- 경기 카드의 3열 구조는 유지해 팀 대 팀의 관계를 깨지 않는다.

## Do’s and Don’ts

### Do

- 첫 화면에서 다음 경기와 현재 상위 선수를 바로 보여준다.
- 오렌지는 핵심 행동과 현재 상태에만 사용한다.
- 점수, 티어, 승률은 고정폭 글꼴로 정렬한다.
- 권한은 서버에서 판정하고 관리자 쓰기 요청은 다시 검증한다.

### Don’t

- 게임 일러스트나 장식으로 경기 정보를 가리지 않는다.
- 18px보다 큰 모서리 반경을 반복 사용하지 않는다.
- 순위 정렬을 수동으로 고정하지 않는다.
- 클라이언트 화면에서만 관리자 기능을 숨기는 것으로 권한 처리를 끝내지 않는다.
